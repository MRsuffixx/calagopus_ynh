## Overview

Calagopus is a modern, open-source game server management panel built with Rust and React. It provides a fast, secure interface for deploying, monitoring, and maintaining game servers — designed for everyone from solo homelabbers to large hosting operators.

Inspired by Pterodactyl but rewritten from scratch in Rust, Calagopus delivers dramatically better performance alongside memory-safety guarantees. The backend uses **axum** as its web framework, **PostgreSQL** via sqlx for persistence, and **Valkey/Redis** via rustis for caching.

### Features

- 🦀 **Rust-powered backend** — over 32,800% throughput improvement vs. PHP-based panels in benchmarks
- ⚡ **Live console** — real-time server output streamed directly in the browser
- 📂 **Full file manager** — browse, upload, download, edit, and extract archives from the web UI
- 📜 **File edit history** — track every change made to server files
- 🗓️ **Scheduled tasks** — time- and event-based automation for server management
- 👥 **Subuser management** — grant scoped access to other users without full admin rights
- 💾 **Advanced backups** — multiple backup drivers with archive and backup browsing support
- 🔐 **WebAuthn authentication** — passkeys, biometrics, and hardware security keys
- 🔗 **OAuth support** — integrate with external identity providers
- 🥚 **Egg system** — compatible with the Pterodactyl ecosystem for arbitrary game support (Minecraft, Rust, ARK, Valheim, FiveM, and more)
- 🌐 **Comprehensive REST API** — every panel action is available via API
- 🔌 **Extension API** — add custom backend logic, routes, UI elements, and database migrations
- 📊 **Role management, activity logs, mount management**, and more

## Disclaimers / important information

### Architecture

This YunoHost package installs the **Calagopus Panel** only. To actually host game servers, you also need at least one **Wings** daemon running on a Linux host (which can be the same machine for a single-node setup). Wings is not bundled in this package.

### First-time setup

After installation, open the panel URL in your browser. You will be greeted by the **Out-Of-Box Experience (OOBE)** wizard — complete it to create your first administrator account and configure the panel before adding Wings nodes.

### Authentication

Calagopus does **not** integrate with YunoHost's LDAP/SSO system. Users are managed entirely within the Calagopus panel itself. The `ldap` and `sso` integration flags are therefore set to `false`.

### Architecture support

This package supports **amd64** and **arm64** only. The Calagopus APT repository does not currently provide `armhf` (32-bit ARM) builds.

### Single instance

Calagopus binds to port 8000 (fixed). Installing more than one instance on the same server is not supported (`multi_instance = false`).

### Resource requirements

- **RAM:** 512 MB minimum at runtime (1 GB recommended; 2 GB if using extensions)
- **Disk:** 1 GB minimum (10 GB recommended if using extensions)

### Environment file

The panel is configured via `/etc/calagopus/.env`. Do not edit this file by hand — YunoHost will overwrite it on upgrade. The `APP_ENCRYPTION_KEY` is generated once at install time and must **never** be changed without re-encrypting all stored secrets.

### Upgrade process

Upgrades are performed via the Calagopus APT repository (`apt update && apt upgrade -y calagopus-panel`) and do not require any manual steps. The panel service will be briefly unavailable during the restart.
